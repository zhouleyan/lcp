## 子资源路由注册
按照目前的代码设计，比如要实现 /namespaces/{namespaceId}/users 的API
我必须定义并实现 memberStorage 作为 namespace 的子资源，最后生成的 API 文档也会将该路由
放到 NamespaceMember下面，这不是我预期的结果

我希望不管几层资源嵌套，比如 /workspace/{workspaceId}/namespaces/{namespaceId}/users ，
只要最后的资源是 User，那么都能通过 UserStorage 的方法注册，且生产的 API 文档，对应的 API 在 User 的组里
