# 主机网络 IP 分配 — 用户故事

> 业务规则：多 IP、手动+自动分配、创建后可增删、移除即解绑（IP 保留占用）、删除主机自动解绑。

## 故事 1：创建主机时选择网络并分配 IP

**作为**平台/工作空间管理员，**我想**在创建主机时选择可用子网并分配 IP，**以便**主机创建后就有明确的网络地址。

**验收标准：**
- 创建主机表单中可以添加多条 IP 配置
- 每条配置：选子网 → 手动输入 IP 或自动分配
- 自动分配时系统从子网取下一个可用 IP
- 手动指定时校验 IP 在子网 CIDR 范围内且未被占用
- 可以为每条 IP 标注用途（management / business / storage 等）
- 子网选择列表只展示有空闲 IP 的公开网络下的子网
- 主机创建和 IP 分配在同一事务中完成，失败则全部回滚
- 不添加任何 IP 也可以创建主机（IP 非必填）

## 故事 2：查看主机的 IP 列表

**作为**管理员，**我想**在主机详情页看到该主机绑定的所有 IP，**以便**了解主机的网络配置。

**验收标准：**
- 主机详情页展示所有绑定的 IP 列表
- 每条记录显示：IP 地址、所属子网名、所属网络名、用途
- 主机列表页也能看到主机绑定的 IP（摘要展示）

## 故事 3：为已有主机追加 IP

**作为**管理员，**我想**给已有主机追加新的 IP，**以便**为主机增加网络连接。

**验收标准：**
- 主机详情页可以发起"添加 IP"操作
- 流程同创建时：选子网 → 手动或自动分配
- 同一子网下可以绑定多个 IP（不同用途）

## 故事 4：从主机移除 IP

**作为**管理员，**我想**从主机移除某个 IP，**以便**调整主机的网络配置。

**验收标准：**
- 主机详情页 IP 列表支持移除操作
- 移除后 IP 仍占用在子网中（解绑但不释放）
- 在网络模块的 IP 分配列表中，该 IP 仍存在但不再关联主机
- 如需彻底回收 IP，管理员到网络模块手动删除该 allocation

## 故事 5：删除主机时自动解绑 IP

**作为**管理员，**我想**删除主机时系统自动解除所有 IP 绑定，**以便**不需要逐个手动移除。

**验收标准：**
- 删除主机不因有绑定 IP 而被阻止
- 删除后所有关联的 IP 自动解绑（host_id 置空）
- IP 仍保留占用在子网中，bitmap 不变
- 批量删除主机同样适用

## 故事 6：网络侧查看 IP 的主机关联

**作为**管理员，**我想**在网络模块的 IP 分配列表中看到每个 IP 绑定了哪台主机，**以便**掌握 IP 的使用情况。

**验收标准：**
- IP 分配列表新增显示：关联主机名、用途
- 未绑定主机的 IP 这两个字段为空（含已解绑的保留 IP）
- 创建 allocation 时也可选择关联到主机（网络侧入口）
